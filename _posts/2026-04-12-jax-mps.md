## Overview

- While [PyTorch](https://github.com/pytorch/pytorch) supports Metal reasonably well, [JAX](https://github.com/jax-ml/jax) has lagged behind.
- A while ago, there was [a post from Apple](https://developer.apple.com/metal/jax/) about a Python package called jax-metal, which added experimental Metal support for JAX.
- The idea was to create a [PJRT](https://openxla.org/xla/pjrt/cpp_api_overview) plugin for [OpenXLA](https://openxla.org/xla/architecture) which maps [StableHLO](https://github.com/openxla/stablehlo) (an [MLIR](https://mlir.llvm.org/) dialect) primitives to their [MPSGraph](https://developer.apple.com/documentation/metalperformanceshadersgraph/mpsgraph) counterparts.
- jax-metal was an interesting proof of concept, but Apple has not invested in keeping it up to date: the latest supported Mac OS version is [Sonoma](https://en.wikipedia.org/wiki/MacOS_version_history#Overview).
- Enter [jax-mps](https://github.com/tillahoffmann/jax-mps), an open-source project inspired by jax-metal. The overall approach is similar, but StableHLO is mapped to [MLX](https://github.com/ml-explore/mlx) instead of MPSGraph.

## How much should you care?

- There is no LLVM backend for Metal, i.e. no equivalent of [NVPTX](https://llvm.org/docs/NVPTXUsage.html) for OpenXLA to target
- Since jax-mps converts StableHLO directly to MLX rather than handing it off to XLA, you don't get the optimizations that JAX's [JIT compliation](https://docs.jax.dev/en/latest/jit-compilation.html) enjoys.
- [It's early days for the project](https://github.com/jax-ml/jax/discussions/34648). Many StableHLO opearations are not yet implemented and the reported speed-up versus CPU is fairly modest, on the order of 3x.
- At this point jax-mps seems more akin to a less mature version of PyTorch's MPS backend rather than a [torch.compile](https://docs.pytorch.org/tutorials/intermediate/torch_compile_tutorial.html) competitor.
- Still, the ability to run existing JAX programs on Metal without porting them to MLX seems promising.

## Install instructions

- Install [uv](https://docs.astral.sh/uv/)
- `uv venv jax-metal --python 3.13`
- `source jax-metal/bin/activate`
- `uv pip install "jaxlib==0.9.0" jax numpy wheel`
- `uv pip install jax-mps`
- `python -c 'import jax; print(jax.devices()); print(jax.numpy.arange(10))'`

If all goes well, you should see output along the following lines:

```
WARNING:2026-04-12 16:44:41,122:jax._src.xla_bridge:905: Platform 'mps' is experimental and not all JAX functionality may be correctly supported!
```
